## 2. CrashReporter

**קובץ:** `packages/support-diagnostics/src/CrashReporter.ts`

### יכולות
- כותב קבצי crash JSON לתיקיית `{dataPath}/reports/crashes/`
- שם קובץ: `crash-{ISO_DATE}-{id}.json`
- מכיל: `id, occurredAt, traceId, source, errorType, message, context, recovered`
- `getRecentCrashes(maxAgeHours)` — מסנן לפי גיל
- מדלג על קבצי JSON שגויים (גנרי — לא קורס)

### מגבלות
- קובץ-מבוסס בלבד — לא DB. אם הדיסק מלא, הכתיבה נכשלת בשקט.
- אין indexing — סריקת קבצים בכל קריאה.

## 3. SystemEvents Table (migration 054)

**שימוש לאחר ביקורת:**
- crash handlers ב-start.ts (מוקדם — לפני startup)
- `POST /api/recovery/event` — כתיבה ידנית מ-C# או מ-dashboard
- RAG worker events
- Startup events מ-DiagnosticsService

### Schema
```sql
CREATE TABLE SystemEvents (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  event_id    TEXT NOT NULL UNIQUE,
  occurred_at TEXT NOT NULL,
  event_type  TEXT NOT NULL,  -- 'crash', 'startup', 'recovery', 'info'
  source      TEXT NOT NULL,  -- 'api', 'desktop', 'agent'
  severity    TEXT NOT NULL CHECK(severity IN ('info','warn','critical')),
  message     TEXT NOT NULL,
  details     TEXT            -- JSON
);
```

### חסרונות
- אין TTL — rows ישנים לא נמחקים אוטומטית.
- מומלץ: מיגרציה 055 — `DELETE FROM SystemEvents WHERE occurred_at < datetime('now', '-90 days')` ב-startup.

## 4. DiagnosticsCollector

**קובץ:** `packages/support-diagnostics/src/DiagnosticsCollector.ts`

### Bundle מכיל
- `SystemSnapshot` — Node version, platform, memory, env vars (redacted)
- `health` — overall status (ok/warn/critical/unknown)
- `recentCrashes` — crash files < 72h
- `recentWarnings` — entries מ-SystemEvents
- `checks[]` — בדיקות runtime

### Chaos Tests (17 בדיקות חדשות)
- malformed JSON → skip
- missing directory → empty array
- network failure → bundle עם unknown status
- invalid JSON response → bundle עם unknown status
- redaction idempotency
- null/undefined handling

## 5. SupportBundleExporter + ZIP

**קובץ:** `packages/api/src/routes/diagnostics.ts`

### תהליך
```
POST /api/diagnostics/bundle
  → collector.collectBundle()
  → exporter.exportToDirectory(tmpDir)
  → zipSingleFile(jsonPath, zipPath)  ← archiver
  → rm(jsonPath)
  → return { bundlePath, filename: 'factum-support-YYYYMMDD-HHMM.zip' }
```

### UI
`SupportExportButton.tsx`:
- מציג שם ZIP (לא path מלא)
- copy-to-clipboard לpath המלא

### חסרונות
- אם `zipSingleFile` נכשל — tmpDir לא נמחק (**חוב GH4**)
- אין הגבלה על גודל bundle (לא נדרש בשלב זה)

## 10. תהליך תמיכה מומלץ

```
1. לקוח מדווח על בעיה
2. לקוח לוחץ "ייצא חבילת תמיכה" ב-/admin
3. ZIP נוצר ← factum-support-YYYYMMDD.zip
4. לקוח שולח ZIP לתמיכה
5. תמיכה מנתחת:
   - bundle.json → system snapshot
   - crashes/*.json → crash history
   - SystemEvents → event timeline
6. תמיכה מחליטה: עדכון / תיקון / שחזור
```
