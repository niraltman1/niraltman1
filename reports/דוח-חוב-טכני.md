## חוב גבוה

### GH1 — WorkflowIdempotencyLog ללא TTL
**קובץ:** `packages/orchestrator/src/orchestrator.ts`
**תיאור:** `acquireLock()` אינו מגדיר expiry. קריסה = נעילה קבועה.
**תיקון:** הוסף `acquired_at` + ניקוי locks ישנים (> 2h) בהפעלה.
**גודל:** S (< 20 שורות).

### GH2 — Zod Validation חסר ב-routes
**תיאור:** חלק מה-routes עושים cast ישיר (`req.body as {}`). injection תיאורטי ל-SQL אינו אפשרי (prepared statements) אך typesafety נפגעת.
**תיקון:** הוסף `z.object({}).parse(req.body)` לכל route.
**גודל:** M (day work).

### GH3 — Rate Limit על AI Endpoints
**תיאור:** `/api/agents/*` כפוף ל-300 req/min גלובלי בלבד. 7 בקשות מקביליות יתפסו את event loop ל-5 דקות.
**תיקון:** `app.use('/api/agents/', rateLimit({ max: 3, windowMs: 60_000 }))`.
**גודל:** XS (2 שורות).

### GH4 — SupportBundleExporter לא מנקה tmpDir בשגיאה
**קובץ:** `packages/api/src/routes/diagnostics.ts`
**תיאור:** אם `zipSingleFile` נכשל, ה-tmpDir נשאר בדיסק.
**תיקון:** `try/finally { await rm(tmpDir, { recursive: true, force: true }) }`.
**גודל:** XS.

### GH5 — Database Backup ללא Integrity Check
**קובץ:** `packages/database/src/hardening.ts`
**תיאור:** `backup()` מבצע `copyFileSync` אך לא בודק שלמות ה-backup.
**תיקון:** פתח `DatabaseConnection` על ה-backup + `PRAGMA integrity_check`.
**גודל:** S.

### GH6 — CORS ב-production מאפשר `localhost:5174`
**קובץ:** `packages/api/src/app.ts`
**תיאור:** port 5174 הוא dev-only (Vite fallback). ב-production — מיותר.
**תיקון:** הסר ב-`NODE_ENV === 'production'` path.
**גודל:** XS.

## חוב בינוני

### BN1 — `console.log` ב-start.ts
כמה הודעות משתמשות ב-`console.log` ישיר ולא דרך pino logger.

### BN2 — `_db` → `_dbHandle` variable naming
שם מ-start.ts — `_db` היה שם לא אינפורמטיבי (תוקן ב-`_dbHandle`).

### BN3 — seedDefaultAdmin בלי first-run check
`seedDefaultAdmin()` נקרא בכל הפעלה. כדאי לבדוק `isFirstRun()`.

### BN4 — אין MigrationRunner timeout
מיגרציה ארוכה (VACUUM גדול) תחסום API startup ללא הודעה.

### BN5 — `getPort` timeout
`get-port` לא מגדיר timeout. בסביבה ממושטרת עלול להתעכב.

### BN6 — agent-core journal צורך prepared statement חדש בכל קריאה
`journalEvent()` מבצע `db.prepare()` בכל קריאה — לא מנוהל cache. עבור pipeline עמוס = overhead.
