## ב. חוזקות אמיתיות

### 1. ארכיטקטורה Local-First — מוצלחת באמת
כל עיבוד הנתונים, כל ה-AI, וכל האחסון מקומיים לחלוטין. אין byte אחד של נתוני לקוחות שיוצא מה-PC. זה לא רק הצהרה — הקוד מאמת זאת: CORS מוגבל ל-localhost, אין HTTP client חוץ לOllama ו-GitHub manifests, audit log בולע את כל ה-routes.

### 2. AI Safety — רמת חשיבה מקצועית
המבנה AI safety stack — circuit breaker, hallucination detector, citation verifier, confidence gate — מדורג גבוה מרוב מוצרי legal-tech. המודל `law-il-E2B:Q4_K_M` מוגן מהחלפה ב-code. הפרדת concern ב-`ai-guardrails` package מאפשרת החלפת guardrails בעתיד.

### 3. Control Plane עם RBAC ו-Idempotency
סוכן שמנסה לרוץ פעמיים על אותו תיק — נחסם. ריצות יתומות מ-crash — מנוקות ב-startup. Case state hash מזהה stale execution. זו רמת הנדסה שמעל הממוצע במוצרי legal SaaS.

### 4. Observability Stack
SystemEvents, AgentExecutionEvents, Metrics, EventStore, dead-letter queue, CrashReporter, RedactionPipeline, SupportBundle ZIP — ביחד יוצרים תמונה מלאה שמאפשרת debug מרחוק. אחרי תיקוני הביקורת — RecoveryPage עובדת עם נתוני אמת.

### 5. כיסוי בדיקות
430+ בדיקות, כולל chaos tests, integration tests, eval regression, stale-execution tests, concurrency stress. מעל הממוצע הענפי למוצר בשלב זה.

### 6. Domain Knowledge עמוקה
54 מיגרציות, 17+ repositories, parsing של 8 פורמטים של מזהים משפטיים ישראליים, כללי פרוצדורה מ-Rules_Engine. המוצר מדבר את השפה של ה-domain.

## ג. חולשות אמיתיות

### 1. OCR חוסם את Event Loop — הסיכון הגדול ביותר
`execFileSync` ב-`ocr-service.ts` חוסם את ה-Node.js event loop לדקות ארוכות בעת עיבוד PDF. API לא מגיב. בסביבת עו"ד יחיד עם PC ייעודי — זה קביל. עם 2+ משתמשים — זה מקצוע. **חייב להיות מטופל לפני הרחבה.**

### 2. אין עדכון אוטומטי
`update-core` package קיים אבל חסר מנגנון הורדה + התקנה. עו"ד צריך לקבל ZIP ולהתקין ידנית. זה לא קביל למוצר מסחרי.

### 3. התקנה ראשונה — חיכוך גבוה
משתמש צריך: (א) להתקין Ollama, (ב) לבצע pull לmodel של 1.8GB, (ג) להגדיר env var, (ד) להפעיל installer. אין wizard.

### 4. Backup רק מקומי
גיבוי שעתי מצוין — אבל רק לכונן מקומי. אם הכונן נכשל, הנתונים אבודים. עבור נתוני לקוחות משפטיים — זה לא קביל.

### 5. 5 באגים קריטיים (תוקנו בביקורת זו)
- `require()` ב-ESM → backup קרסה בשקט
- עמודות SQL שגויות ב-recovery routes → dashboard ריק תמיד
- tabularRouter לא נרשם → 404 על כל import CSV/Excel
- crash handlers אחרי startup → קריסות startup לא נרשמות
- stale locks לא מנוקות → agents חסומים לצמיתות אחרי crash

## ד. מוכנות מסחרית — לפי פלח

### עורך דין יחיד, PC ייעודי, Windows 10/11

**כן, ניתן לפרוס עכשיו. עם אזהרות:**
- נדרשת התקנה ידנית (Ollama + pull + env var)
- OCR של PDF גדול יעצור את הUI עד כמה דקות — לתעד כ-known behavior
- עדכונים ידניים
- גיבוי מקומי בלבד

**ציון מוכנות:** 7 / 10

### משרד קטן 2-5 עו"ד

**לא מומלץ עדיין.**
- OCR blocking גורם ל-starvation
- אין multi-user auth (כולם עם אותו admin)
- DB לא ממוטב ל-concurrent writes

**ציון מוכנות:** 4 / 10

### משרד בינוני 10+ עו"ד

**לא.**
- ארכיטקטורה לא מתאימה
- SQLite לא מתאים ל-10+ concurrent users
- אין audit trail מלא
- אין AD integration

**ציון מוכנות:** 1 / 10

## ה. מה עוצר release אמיתי

### חוסם 1 — עדכון אוטומטי
ללא OTA update, כל עדכון דורש ביקור IT. זה מונע scaling.
**פתרון:** OTA downloader + verifier + installer (NSIS/WiX)
**עלות:** ~60h

### חוסם 2 — OCR blocking
ללא worker_threads, כל PDF גדול = API frozen.
**פתרון:** מעבר ל-worker_threads
**עלות:** ~40h

### חוסם 3 (חלקי) — Onboarding Wizard
בלי wizard, אין installation self-service. נדרש IT.
**פתרון:** /setup page + Ollama pull with progress
**עלות:** ~20h

### חוסם 4 (חלקי) — Backup External
לקוח עם כונן אחד — נתונים חשופים לאובדן.
**פתרון:** NAS target / USB target / OneDrive
**עלות:** ~20h
