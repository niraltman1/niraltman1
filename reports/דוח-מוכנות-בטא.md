## מה אינו יציב

### ⚠️ OCR חוסם Event Loop
`execFileSync` ב-OCRService חוסם את ה-event loop. PDF גדול (30+ עמודים) = API מושהה לכל שאר המשתמשים. **בסביבת עו"ד יחיד — קביל. בצוות — בעיה.**

### ⚠️ WorkflowIdempotencyLog ללא TTL
נעילות תהליך שקרסו יכולות להישאר לצמיתות. דורש manual clear דרך `/admin/recovery`.

### ⚠️ חוסר Update Mechanism
`update-core` package קיים (VersionManifestParser, UpdateChannelManager) אך אין מנגנון הורדה + התקנה אוטומטי. עדכונים דורשים התקנה ידנית.

### ⚠️ Ollama חייב להיות מותקן מראש
המשתמש צריך להתקין Ollama + pull model (~1.8GB) לפני הפעלה ראשונה. אין installer guide ב-UI.

## מה חוסם בטא אמיתי

### 🔴 חוסם 1: עדכונים אוטומטיים
אין מנגנון עדכון שלם. עו"ד לא יכול לעדכן ב-1-click. דורש zip + install ידני.

### 🔴 חוסם 2: OCR הוא single-threaded blocking
פלטפורמה ב-multitenant (ולו 2 עו"ד באותו PC) תסבול מ-starvation בזמן OCR.

### 🟡 חוסם חלקי 3: אין מדריך התקנה מלא
installer PowerShell קיים אך לא נבדק ב-clean machine.

### 🟡 חוסם חלקי 4: Backup ל-external storage
גיבוי מקומי בלבד. אם הכונן נכשל, נתוני הלקוחות אבודים.
