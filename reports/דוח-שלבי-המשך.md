## שלב GA (מחזור 2)

**יעד:** מוצר מסחרי ל-Windows עם עדכון אוטומטי + onboarding

### GA1 — OTA Update (P1, גדול XL)
```
update-core (קיים):
  ├── VersionManifestParser ✅
  ├── UpdateChannelManager ✅
  └── (חסר) UpdateDownloader
       ├── download .zip → %TEMP%
       ├── verify SHA-256
       ├── extract → staging dir
       └── swap + restart

installer (נדרש):
  └── NSIS / WiX → MSI עם auto-update
```

### GA2 — Onboarding Wizard
```
/setup (new route):
  Step 1: בדיקת Ollama
  Step 2: pull model (progress bar)
  Step 3: הגדרת תיקיית עבודה
  Step 4: import ראשון (optional)
  Step 5: FirstRun complete
```

### GA3 — Backup ל-External Storage
```typescript
// packages/encrypted-backup/src/external-target.ts (new)
interface BackupTarget {
  upload(snapPath: string): Promise<void>;
}
// implementations: NasTarget, OneDriveTarget, UsbTarget
```

### GA4 — SystemEvents TTL (מיגרציה 055)
```sql
-- בהפעלה: ניקוי events ישנים מ-90 יום
DELETE FROM SystemEvents WHERE occurred_at < datetime('now', '-90 days');
```

### GA5 — CSP hardening (הסר unsafe-inline)
- מעבר ל-nonces ב-CSP
- `script-src 'nonce-{random}'` במקום `'unsafe-inline'`

## שלב Enterprise (מחזור 3)

**יעד:** משרד עם 10-50 עו"ד, IT, audit trail

### E1 — Multi-User Architecture
```
כעת: SQLite local, 1 connection
Enterprise: SQLite per-user (read-only shared schema)
            OR PostgreSQL migration (bigger lift)
```

### E2 — Active Directory / LDAP Auth
- SAML/OIDC integration
- GPO deployment
- Role sync מ-AD groups

### E3 — Audit Trail מלא
- כל פעולה של כל user נרשמת
- Export לPDF לרגולציה
- Immutable log (append-only)

### E4 — Multi-Office / Multi-DB
- כל סניף = DB נפרד
- Central management dashboard
- Replication (SQLite → SQLite WAL-based)

### E5 — API v2 (REST → GraphQL optional)
- לא נדרש, אך enterprise clients דורשים לפעמים
