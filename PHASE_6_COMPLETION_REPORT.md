## Features Added

### 1. Architecture Guard (`scripts/check-architecture.ts`)

A Node.js script that scans route files and package sources for architectural violations.

**Critical rules (exit 1, blocks CI):**
- `NO_DIRECT_SQL_IN_ROUTES` — `db.prepare()` calls in `packages/api/src/routes/*.ts`
  (allowlisted files: `diagnostics.ts`, `setup.ts`, `updates.ts`, `citations.ts`, `communications.ts`)
- `NO_DASHBOARD_IMPORT_FROM_PACKAGES` — any `packages/*` source importing from `apps/dashboard`

**Warning rules (exit 0, report only):**
- `ROUTE_TOO_LONG` — route files exceeding 120 non-blank lines
- `ARRAY_CHAIN_IN_ROUTE` — 3+ `.map/.filter/.reduce` chains inside a route file

Outputs `ARCHITECTURE_AUDIT.md` at the repo root after every run.

### 2. Repository Coverage Generator (`scripts/generate-repo-coverage.ts`)

Scans `packages/database/src/queries/` to discover all Repository classes, infers which
database tables each accesses (via SQL string regex), and checks import coverage across
the entire codebase. Outputs `REPOSITORY_COVERAGE.md` at the repo root.

- Found **35** repository classes across 34 query files
- Coverage: 416 source files scanned for import references

### 3. API Contract Snapshot Tests (`packages/api/src/__tests__/api-contract.test.ts`)

Vitest snapshot tests for 3 core routes, verifying response shape stability:
- `GET /api/cases` — top-level response keys snapshot
- `GET /api/entities/judges` — entity list shape snapshot
- `GET /api/updates/health` — update health response shape snapshot

Tests follow the established pattern in `packages/api/src/routes/__tests__/` (in-memory SQLite,
mocked repositories, supertest HTTP).

### 4. CI Integration (`.github/workflows/ci.yml`)

Added `Architecture guard` step to the `check:` job (ubuntu-latest), running after `Lint` and
before `Test (logic — all unit tests)`:

```yaml
- name: Architecture guard
  run: pnpm check:arch
```

### 5. Root Package Script

Added `"check:arch": "tsx scripts/check-architecture.ts"` to root `package.json`.

### 6. Dependency Audit (`DEPENDENCY_AUDIT.md`)

New document with the complete internal dependency graph for all 24 packages, layered
architecture diagram, and the architectural constraints enforced by the guard.

## Architecture Violations Found

The architecture guard found **151 critical violations across 23 route files** on the current
codebase. These represent direct `db.prepare()` SQL calls in route handlers that should be
refactored into Repository methods.

Key offending routes (by violation count):
- `admin.ts` — 11 direct SQL calls, 472 LOC (also exceeds LOC limit)
- `entities.ts` — multiple db.prepare() calls for graph queries
- `agents.ts`, `activity.ts` — direct SQL in route handlers

Additionally, **31 warnings** were found:
- 29 route files exceed the 120 non-blank line limit
- Multiple files have 3+ chained array operations inside route handlers

See `ARCHITECTURE_AUDIT.md` for the full violation table.

> **Note:** The CI `Architecture guard` step will currently **fail** on the existing codebase
> due to these pre-existing violations. Resolving them (by extracting SQL into Repository
> methods) is tracked as a follow-up task (Phase 7 or a dedicated refactoring sprint).
> The CI gate can be temporarily set to `continue-on-error: true` if needed to unblock
> other work while violations are being addressed.
