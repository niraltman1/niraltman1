## 2. שכבת הנתונים

### מסד הנתונים
- **SQLite + better-sqlite3** — סינכרוני, WAL mode, foreign keys ON
- **2 קבצי DB**: `factum-il.db` (main) + `factum-il_data.db` (ATTACH AS data_store)
- **54 מיגרציות** — רצות אוטומטית בהפעלה
- **cache_size = 32MB**, `busy_timeout = 30s`, `synchronous = NORMAL`

### מודול ה-DB (`@factum-il/database`)
| מחלקה | תפקיד |
|-------|--------|
| `DatabaseConnection` | חיבור יחיד, WAL, ATTACH |
| `MigrationRunner` | הרצת מיגרציות + checksum validation |
| `DatabaseHardening` | WAL checkpoint, integrity_check, backup |
| `SearchEngine` | FTS5 search + Hebrew tokenization |
| 19+ Repositories | ClientRepository, CaseRepository, ... |

        ├── enrich() — 45s timeout, circuit breaker
        └── stream() — SSE streaming

  └── CircuitBreaker per model
        ├── threshold: 5 failures → OPEN
        ├── cooldown: 30s

## 5. שכבת ה-Pipeline (OCR)

```
@factum-il/pipeline
  └── OCRService
        ├── Lane 1: OCRmyPDF (deskew + rotate, highest quality)
        ├── Lane 2: pdftotext (already-digital PDFs)
        ├── Lane 3: Docling (layout-aware, tables, multi-column)
        └── Lane 4: Ghostscript + Tesseract (last resort)
  └── PipelineSupervisor — file watcher + queue management
  └── LockService — document-level mutex
```

**שפות OCR:** `heb+eng` ברירת מחדל, DPI=300, PSM=6

## 7. שלד ה-Desktop (C#/WPF)

```
FactumIL.Desktop/
├── App.xaml.cs           ← OnStartup, RunBootSequenceAsync
├── ApiHostService.cs     ← spawn Node.js API process
├── OllamaService.cs      ← spawn + health-check Ollama
├── StartupValidator.cs   ← 7 בדיקות הפעלה
├── RecoveryWindow.xaml.cs ← חלון שחזור בעברית
├── SplashWindow.xaml.cs  ← splash עם progress
├── DiagnosticsService.cs ← crash capture + startup snapshot
└── MainWindow.xaml.cs    ← WebView2 window
```

### רצף האתחול
```
1. OnStartup
   a. InstallGlobalHandlers (crash capture)
   b. ApiHostService.Start() — spawn Node.js
   c. OllamaService → check installed + warn if not
   d. SplashWindow.Show()
2. RunBootSequenceAsync (background task)
   a. OllamaService.StartAsync()
   b. WaitForApiAsync (30s timeout)
   c. OllamaService.WaitForReadyAsync + EnsureModelAsync
   d. StartupValidator.ValidateAsync() — 7 checks
   e. If errors → RecoveryWindow (modal)
   f. If user continues → restart API in safe mode
   g. MainWindow.Show()
```
