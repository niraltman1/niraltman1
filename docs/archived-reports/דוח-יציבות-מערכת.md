# דוח יציבות מערכת — Factum-IL

> נוצר: 2026-05-26

---

## ציון יציבות כולל: **7.8 / 10**

---

## ניתוח לפי רכיב

### מסד הנתונים — 9/10

**חוזקות:**
- WAL mode + `busy_timeout = 30s` — מנוח race conditions בין writes
- `PRAGMA synchronous = NORMAL` — איזון בין ביצועים לאמינות
- Foreign keys ON — integrity referential
- `integrity_check` + `foreign_key_check` — ניתן להפעלה
- MigrationRunner עם checksum — לא ניתן לשנות מיגרציה שיצאה
- 54 מיגרציות, כל אחת ב-transaction אוטומטי
- `SKIP_ON_ERROR` flag — migration 052 (sqlite-vec) לא קורסת אם הextension חסר

**חולשות:**
- חיבור DB יחיד — לא מסוכן ב-SQLite אך בעומס גבוה ייתכן contention
- Backup ללא integrity verification לאחר copy

**כשל מדומה:**
- DB corruption: `integrity_check` מזהה → API מוסיף שורת crash ל-SystemEvents → RecoveryWindow מתריע
- DB locked: 30s timeout → error → crash handler

---

### AI / Ollama Layer — 8/10

**חוזקות:**
- Circuit breaker: 5 failures → OPEN, 30s cooldown
- Timeout: connect 5s, request 45s
- `isAvailable()` health check לפני כל enrichment
- Guardrails: hallucination + citation + confidence gate
- Graceful degradation — pipeline ממשיכה ללא AI אם Ollama down

**חולשות:**
- אם circuit breaker פתוח, RAG worker לא מנסה שוב עד restart
- אין retry logic עם backoff ב-OllamaClient
- 45s timeout = event loop תפוס לנצח אם Ollama תקוע (ולא מגיב)

**כשל מדומה:**
- Ollama crash → circuit breaker OPEN → all AI requests fail gracefully → documents processed without enrichment
- Ollama timeout → AbortSignal.timeout(45000) → error propagated → crash NOT expected

---

### Pipeline (OCR) — 5/10

**חוזקות:**
- 4 lanes OCR — Lane 1 נכשל → Lane 2 → Lane 3 → Lane 4
- Cache OCR results (file hash) — לא מעבד אותו קובץ פעמיים
- tmpDir cleanup ב-`finally`
- Supervisor עם file watcher

**חולשות:**
- `execFileSync` = blocking — **הגדול ביותר ביציבות**
- BATCH_SIZE=3 concurrent OCR processes = API frozen עד 3 דקות
- אין retry על OCR failure עם backoff
- OCRmyPDF + Tesseract חייבים להיות מותקנים — אין validation ב-startup

**כשל מדומה:**
- 10 PDFs בתור: יעובדו עד 3 בו-זמנית, אך API יהיה frozen בזמן זה
- OCRmyPDF לא מותקן: Lane 1 fails → Lane 2 runs → ממשיך

---

### Agent Layer — 8/10

**חוזקות:**
- `INSERT OR IGNORE` + UNIQUE constraint — אין duplicate runs
- `canRunAgent()` — atomic check
- Stale detection (10 min)
- Journal events ל-AgentExecutionEvents
- RBAC + case execution guard
- Cleanup אוטומטי ב-startup (תוקן בביקורת זו)
- `markAgentFailed()` ב-catch — לא נשאר running אם agent קורס

**חולשות:**
- Agent inference ב-main thread (45s blocks event loop)
- אין timeout על agent ריצה שלמה (רק על Ollama request)

---

### Recovery / Observability — 8/10

**חוזקות (לאחר ביקורת):**
- crash handlers רשומים לפני startup
- SystemEvents נכתב גם בקריסות startup
- stale lock cleanup אוטומטי
- RecoveryPage עם נתוני-אמת (תוקן — עמודות נכונות)
- clear-locks עובד (תוקן — `finished_at` ולא `completed_at`)
- SupportBundle ZIP עם redaction

**חולשות:**
- אין alerting active (אין email/SMS אם המערכת קורסת)
- SystemEvents לא מוחק rows ישנים אוטומטית (table יגדל)

---

### Desktop Shell (C#) — 7/10

**חוזקות:**
- 7-step StartupValidator
- RecoveryWindow — מאפשר continue ב-safe mode
- DiagnosticsService — crash capture גם מ-C# side
- OllamaService — warning Hebrew אם לא מותקן

**חולשות:**
- אין test coverage ל-C# code
- `WaitForApiAsync` — 30s timeout, אחר כך MessageBox (לא graceful enough)
- API port hardcoded 3001 — אם port תפוס, ApiHostService לא מעדכן את MainWindow URL

## תרחישי כשל ותגובת המערכת

| תרחיש | תגובה | ציון |
|-------|--------|------|
| DB corruption | integrity_check מזהה → crash event → RecoveryWindow | ✅ |
| DB locked > 30s | SQLite error → Express 503 | ✅ |
| Ollama crash | Circuit breaker → AI skip → pipeline ממשיכה | ✅ |
| uncaughtException | Crash event ל-SystemEvents → shutdown | ✅ (לאחר תיקון) |
| Migration failure | API לא עולה → RecoveryWindow | ✅ |
| OCR קורס | Lane fallback | ✅ |
| Agent duplicate | INSERT OR IGNORE → 409 | ✅ |
| Agent orphan (crash) | Cleanup בהפעלה הבאה | ✅ (לאחר תיקון) |
| Disk full | Backup נכשל → warning | ⚠️ |
| Node.js < 20 | FATAL exit עם הודעה | ✅ |
| port 3001 תפוס | get-port מחפש 3001-3021 | ✅ |
| FACTUM_IL_ROOT חסר | StartupValidator error → RecoveryWindow | ✅ |

## מסקנה

המערכת יציבה מספיק לסביבת **עו"ד יחיד** עם:
- PC ייעודי
- Windows 10/11
- Ollama מותקן עם המודל הנכון
- לפחות 16GB RAM + 50GB disk

**הסיכון הגדול ביותר ליציבות** הוא OCR blocking ב-main thread — נסביר כ-known limitation עד GA.
