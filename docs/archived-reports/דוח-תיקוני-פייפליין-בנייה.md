### PR #32 — `--ignore-scripts` + `npm rebuild` (תיקון סופי)

| | |
|---|---|
| **קובץ** | `publish.ps1` שלב 8.5 |
| **שגיאה** | `[ERR_PNPM_IGNORED_BUILDS] Ignored build scripts: better-sqlite3@9.6.0` |
| **גורם** | pnpm 10+ חוסם הרצת build scripts של native modules אלא אם מאושרים במפורש. pnpm קורא את ההגדרה רק מ-ROOT workspace — לא מהתיקייה המבודדת שנוצרת בבנייה. בנוסף: `better-sqlite3@9.6.0` (גרסה ישנה) אין לה Node-22 prebuilt binary — אפילו אם ה-build הותר, הקישור הנטיבי היה נכשל בזמן ריצה. |
| **תיקון** | עקיפת pnpm לחלוטין: `pnpm install --prod --ignore-scripts` → `npm rebuild better-sqlite3`. `overrides: better-sqlite3: "^11.0.0"` ב-`pnpm-workspace.yaml` מבטיח גרסה עם Node-22 prebuilt binary. self-verification: `node -e "require('better-sqlite3')"` בסוף שלב 8. |
| **מדוע גרסה ^11** | הריפו כבר מצביע ל-`11.10.0` ב-lockfile; Node-22 prebuilt binary קיים רק מגרסה 11+ |

### PR #36 — תיקון נתיב 8.3 ב-`$env:TEMP`

| | |
|---|---|
| **קובץ** | `publish.ps1` שלב 10 (הורדת Node.js) |
| **שגיאה** | `Remove-Item : An object at the specified path C:\Users\021A~1 does not exist.` |
| **גורם** | שם המשתמש עברי (`ניר`) גורם ל-Windows לייצר שם קצר 8.3 עבור ה-profile directory. `$env:TEMP` מחזיר `C:\Users\021A~1\AppData\Local\Temp` — נתיב 8.3 קצר. `Remove-Item` עם `$ErrorActionPreference = 'Stop'` נכשל בפתרון הנתיב הקצר, במיוחד כשהתיקייה קיימת מריצה קודמת. |
| **תיקון** | `$TempDir = (Get-Item -LiteralPath $env:TEMP).FullName` — ממיר נתיב 8.3 לנתיב ארוך מלא (`C:\Users\ניר\AppData\Local\Temp`). בנוסף: `-ErrorAction SilentlyContinue` על `Remove-Item` למניעת קריסה על תיקייה שיורית |
| **הערה** | בעיה ייחודית לסביבות Windows עם שמות משתמש בשפות non-ASCII (עברית, ערבית, יפנית וכו') |

### PR #37 — תיקון `faDirectory` ב-`installer.iss`

| | |
|---|---|
| **קובץ** | `installer.iss` שורה 199 (`[Code]` section) |
| **שגיאה** | `Error on line 199: Column 39: Unknown identifier 'faDirectory'. Compile aborted.` |
| **גורם** | הפונקציה `IsDotNet8DesktopInstalled()` כתובה בסגנון Delphi/SysUtils: `FindFirst(path, faDirectory) <> 0`. אך ב-Inno Setup Pascal Script: (1) `faDirectory` לא קיים, (2) `FindFirst` מקבל `var FindRec: TFindRec` (לא דגל attributes), (3) `FindFirst` מחזיר `Boolean` (לא `Integer` שבו `<> 0` = כשל) |
| **תיקון** | הוספת `FindRec: TFindRec` למשתני הפונקציה + שכתוב הבלוק: `if FindFirst(path, FindRec) then begin FindClose(FindRec); Result := True; Exit; end;` |
| **השפעה** | ISCC מקמפל את ה-`[Code]` section בהצלחה → `Factum-IL-Setup.exe` נוצר |

## לקחים

1. **pnpm 10+ אינו קורא `pnpm` field מ-`package.json`** — הגדרות `onlyBuiltDependencies` חייבות להיות ב-`pnpm-workspace.yaml` של ה-ROOT, ולא ב-workspace המבודד שנוצר ב-staging.
2. **`--ignore-scripts` + `npm rebuild` עוקף את כל מנגנון האישור** — פתרון פשוט ויציב יותר ממאבק עם הגדרות workspace.
3. **שמות משתמש עבריים יוצרים נתיבי 8.3** — כל שימוש ב-`$env:TEMP`, `$env:USERPROFILE`, `$env:APPDATA` על מכונה עם שם משתמש עברי דורש `(Get-Item -LiteralPath ...).FullName`.
4. **Inno Setup Pascal ≠ Delphi SysUtils** — `FindFirst`, `FindNext`, `FindClose` בינוסטאלר עובדים עם `TFindRec`, לא עם דגלי attributes של Delphi.
5. **`AssemblyName` ב-csproj קובע את שם ה-exe** — חייב להתאים ל-`AppExeName` ב-`installer.iss`.
