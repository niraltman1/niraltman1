# NOTIFICATIONS-INBOX-IMPLEMENTATION-PLAN

> **STATUS: IMPLEMENTED**
> All features described in this plan are shipped and live in v1.0.0.
> Date implemented: May 2026

---

## Summary

The Notifications inbox is fully implemented. Users receive in-app notifications for document processing events, agent completions, deadline warnings, and system alerts.

## What Was Built

- `Notifications` table (migration 029) — inbox per user
- Notification types: `document_processed`, `agent_complete`, `deadline_warning`, `system_alert`, `backup_complete`, `fts_repaired`
- Unread badge in top navigation bar (auto-refreshes every 30s)
- Inbox slide-over panel with mark-as-read and clear-all
- Notification generated by pipeline on `VERIFIED` state
- Notification generated by orchestrator on agent completion
- Notification generated by `litigation-intelligence` for deadline risk (≥ 0.7 risk score)
- Notification generated by backup scheduler on encrypted backup failure

## Schema

```sql
Notifications (
  id          INTEGER PRIMARY KEY,
  user_id     INTEGER,
  type        TEXT NOT NULL,
  title_he    TEXT NOT NULL,
  body_he     TEXT,
  link        TEXT,          -- optional deep link route
  is_read     INTEGER DEFAULT 0,
  created_at  TEXT DEFAULT (datetime('now'))
)
```

## API Endpoints

```
GET  /api/notifications          → Notification[] (unread first)
POST /api/notifications/:id/read → mark single as read
POST /api/notifications/read-all → mark all as read
DELETE /api/notifications/:id    → delete notification
```
